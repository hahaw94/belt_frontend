🎯 问题：
   - HTTP-FLV 无法播放 ❌
   - HLS 延迟太高（2-10秒）⚠️

✅ 解决方案：

--------------------------------------------------------------------------------
方案 1：WebSocket-FLV（最推荐）⭐⭐⭐⭐⭐
--------------------------------------------------------------------------------

延迟: <1秒
URL: ws://10.18.21.219:18081/rtp/34020000001320000001_34020000001320000001.live.flv?originTypeStr=rtp_push

操作:
  1. 刷新页面（Ctrl+Shift+R）
  2. 点击 "WS-FLV (推荐)" 按钮
  3. 点击播放

原因:
  ✅ 您的本地流已验证成功（ws://localhost:18081/live/...）
  ✅ WebSocket 长连接，数据推送更稳定
  ✅ 避开 HTTP-FLV 的兼容性问题

预期日志:
  ✅ [FLV] MEDIA_INFO
  ✅ [Video] loadeddata
  ✅ readyState=3 或 4
  ✅ 播放成功

--------------------------------------------------------------------------------
方案 2：WebSocket-FMP4 ⭐⭐⭐⭐
--------------------------------------------------------------------------------

延迟: <1秒
URL: ws://10.18.21.219:18081/rtp/34020000001320000001_34020000001320000001.live.mp4?originTypeStr=rtp_push

操作:
  1. 点击 "FMP4-WS" 按钮
  2. 点击播放

优势:
  ✅ 更现代的 MP4 封装格式
  ✅ 可能比 FLV 兼容性更好

--------------------------------------------------------------------------------
方案 3：HTTP-FMP4 ⭐⭐⭐
--------------------------------------------------------------------------------

延迟: 1-2秒
URL: http://10.18.21.219:18081/rtp/34020000001320000001_34020000001320000001.live.mp4?originTypeStr=rtp_push

操作:
  1. 在输入框中输入上述 URL
  2. 点击播放

说明:
  如果 WebSocket 有问题，尝试此方案

--------------------------------------------------------------------------------

📊 延迟对比：

  WebSocket-FLV  :  <1秒   ⭐⭐⭐⭐⭐ 最推荐
  WebSocket-FMP4 :  <1秒   ⭐⭐⭐⭐
  HTTP-FMP4      :  1-2秒  ⭐⭐⭐
  HLS            :  2-10秒 ⭐ (延迟高)
  HTTP-FLV       :  不可用  ❌

--------------------------------------------------------------------------------

🔧 如果 WebSocket 连接失败：

检查防火墙:
  netsh advfirewall firewall add rule name="WVP WebSocket" dir=in action=allow protocol=TCP localport=18081

检查 WVP 是否运行:
  tasklist | findstr MediaServer

重启 WVP:
  taskkill /F /IM MediaServer.exe
  cd D:\study\belt_frontend\front\smart-video-platform\Release
  start MediaServer.exe

--------------------------------------------------------------------------------

💡 成功标志：

  ✅ 看到 "[FLV] MEDIA_INFO"
  ✅ readyState 变为 3 或 4
  ✅ 视频画面显示
  ✅ 延迟 <1秒

失败标志：

  ❌ readyState 一直是 0
  ❌ 没有 "[FLV] MEDIA_INFO"
  ❌ 下载速度 0.01 KB/s
  ❌ 控制台报错

--------------------------------------------------------------------------------

🎯 立即测试（30秒）：

  1. 刷新页面
  2. 点击 "WS-FLV (推荐)"
  3. 点击播放
  4. 观察日志

很可能直接成功！🚀
